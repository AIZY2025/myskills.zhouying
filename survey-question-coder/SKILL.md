# survey-question-coder · 问卷题号化 Skill

为问卷原始数据的工作簿生成一份"带题号"的"整理"sheet,方便后续做频率分析、交叉表、可视化等。

> **典型工作流**:在「原始」sheet 基础上,生成「整理」sheet,A 列是"整理"标识(值=1),B 列起每列在第 1 行带题号(G1/H3-1/K1-2 等),第 2 行保留原表头做对照,第 3 行起是原数据。

## 适用场景

- 用户说:"给问卷原始数据每个题目一个编号" / "把题号加到表头" / "原始表整理成题号化" / "原始 sheet 整理 sheet 题号化"
- 用户提供了"原始"sheet 风格的工作簿(列头含【XX】题号或多列同题),需要程序化生成"整理"sheet
- 需要批量为多份问卷/多 sheet 加题号

## 不适用

- 工作簿里**没有"原始"sheet**——脚本会报错,需要先确认 sheet 名
- 题号已经在工作簿里写好了(用户只想"查看"题号,不需要本 skill)
- 用户希望按**自定义字母序号**(G/H/I/J/K 顺序递增)而不是【XX】号——本 skill 严格保留原题头里的【XX】号

## 题号规则(核心)

| 表头形式 | 提取规则 | 例子 |
|---|---|---|
| `数字.【XX】xxx` | 优先用【XX】(即字母+数字) | `2.【G1】贵机构...` → `G1` |
| `数字.中文` (无【】) | 用 `Q` + 数字 | `1.第一部分...` → `Q1`;`15.您还有...` → `Q15` |
| 其它(无任何题号) | `Q` + 递增序号(从已用 Q+数字 的最大值 +1 开始) | `编号` → `Q16`;`IP` → `Q27` |

**多列同题**:同一题号对应多列时,加 `-1`/`-2`/`-3` 后缀(单列不加):

- `2.【G1】:A` / `2.【G1】:B` / ... → `G1-1` / `G1-2` / ... / `G1-5`
- `1.第一部分_填空1` ~ `_填空4` → `Q1-1` / `Q1-2` / `Q1-3` / `Q1-4`
- `6.【H3】:A` ~ `:D` → `H3-1` / `H3-2` / `H3-3` / `H3-4`
- `13.【K1】:A` ~ `:J` → `K1-1` / `K1-2` / ... / `K1-10`

## 输出结构(参考"整理"sheet)

```
A1 = 'Q'              ← 题号列标题
A2 = '整理'            ← 标识列列名
A3.. = 1              ← 标识列数据(全部 1)

B1 = 'Q16'            ← 编号列的题号
B2 = '编号'            ← 原表头
B3.. = 301, 300...    ← 原数据(从原"原始"sheet 的行 2 起)

C1 = 'Q17'            ← 开始答题时间的题号
C2 = '开始答题时间'
C3.. = 2026-06-29 07:58:38, ...

F1 = 'Q1-1'           ← 1.第一部分_填空1 的题号
F2 = '1.第一部分：基本信息_填空1'
F3.. = '晨韵教育培训', ...

J1 = 'G1-1'           ← 2.【G1】 第 1 个选项的题号
J2 = '2.【G1】...:持有教育局颁发的办学许可证'
J3.. = 'A.持有...', ...
...
```

## 用法

### 方式 1:命令行(直接跑脚本)

```bash
python scripts/code_survey_questions.py <input.xlsx> [-o <output.xlsx>]
```

参数:
- `input` (必填):输入 .xlsx(必须含「原始」sheet)
- `-o` / `--output`:输出 .xlsx 路径(默认在 input 同目录下生成 `<原名>_题号化.xlsx`)
- `--source-sheet`:原始 sheet 名(默认 `原始`)
- `--out-sheet`:输出 sheet 名(默认 `整理`)
- `--mark-col-header`:A1 标题(默认 `Q`)
- `--mark-col-label`:A2 列名(默认 `整理`)
- `--mark-col-value`:A 列除标题/列名外的填充值(默认 `1`)

示例:

```bash
python scripts/code_survey_questions.py "/Users/admin/Desktop/测试/【分析】机构端数据图表分析-公式版.xlsx"
# 会在同目录生成「【分析】机构端数据图表分析-公式版_题号化.xlsx」
```

### 方式 2:Agent 直接调用脚本

当用户的需求是用本 skill 跑一份问卷数据时,直接:

1. 确认输入 .xlsx 存在,且包含「原始」sheet
2. 跑 `python <skill_dir>/scripts/code_survey_questions.py <input.xlsx> -o <output.xlsx>`
3. 用 `openpyxl` 抽查输出:第 1 行是题号、第 2 行是原表头、A3..=1
4. 把输出 .xlsx 路径交给用户
5. **结束后提示**:告知用户「如果需要对这个题号化后的数据进行交叉分析，请运行 /cross-analysis-tables」

### 方式 3:作为 Python 模块 import

```python
from code_survey_questions import build_question_codes, write_coded_sheet
import openpyxl

wb = openpyxl.load_workbook("input.xlsx")
src = wb["原始"]

# 只看题号(不写文件)
codes = build_question_codes(src)   # [(col_idx, header, final_code), ...]
for c, h, code in codes[:5]:
    print(code, "<-", h)

# 写入新 sheet
out_ws = write_coded_sheet(src, wb, out_sheet_name="整理")
wb.save("output.xlsx")
```

- `Q16` 编号 / `Q17` 开始答题时间 / `Q18` 结束答题时间 / `Q19` 答题时长
- `Q1-1`~`Q1-4` 1.第一部分 4 个填空
- `G1-1`~`G1-5` 2.【G1】5 个选项
- `G2-1`~`G2-7` 3.【G2】7 个选项
- `H1` 4.【H1】 / `H2` 5.【H2】 / `H3-1`~`H3-4` 6.【H3】4 个选项
- `I1`~`I5` 7~11 单选 / `I6-1`~`I6-6` 12.【I6】6 个选项
- `K1-1`~`K1-10` 13.【K1】10 个选项 / `K2-1`~`K2-9` 14.【K2】9 个选项
- `Q15` 15.您还有...
- `Q20`~`Q29` 语言/IP/UA/Referrer 等元数据(无题号,递增)

## 已知边界 / 注意事项

1. **"1.第一部分" 4 个填空的归类**:4 个相邻列的表头都是 `1.第一部分:基本信息_填空N` 格式,提取数字都是 `1`,所以共享 `Q1` 并加 `-1`/`-2`/`-3`/`-4` 后缀。这**符合用户的"填空合并"语义**,但**与已有的"整理"表不直接一致**——已有的"整理"表把这 4 列丢弃了。本 skill 按"全部保留"处理,题号 `Q1-1`~`Q1-4` 是新加的。

2. **元数据列(编号/开始/结束/时长/语言/IP/UA/Referrer/地理位置等)**:用户明确要求"全部编题号保留",因此**全部参与编码**,会得到 `Q16`~`Q29` 这类连续编号。如果用户希望丢弃这些列,需要后续在"整理"表里手工删。

3. **无前缀冲突保护**:脚本会先扫描所有"数字.中文"占用的 Q+数字,让"X 模式"(无题号)的递增从 `max+1` 开始,避免与 `Q1`/`Q2`/`Q15` 冲突。例如 1.第一部分 占 `Q1`,则"编号"列(列 1)得到 `Q16`(16 = 15+1),不会撞 `Q1`。

4. **空列自动跳过**:原表里有空白列(表头为 None)会被跳过,不参与编码。

5. **不修改原表**:`write_coded_sheet` 不会触碰"原始"sheet,只在工作簿里新增一个"整理"sheet(若已存在会先删再建)。

6. **公式被原样复制**:如果原表里有公式(如 SUM/IF),脚本会复制公式字符串本身(不计算结果)。若需要结果数据,用 `data_only=True` 重新打开再读。

## 调试技巧

- 用 `openpyxl.load_workbook("output.xlsx")` 打开输出
- `ws["整理"]` 拿到整理表
- `ws.cell(1, c).value` 拿题号,`ws.cell(2, c).value` 拿原表头
- 重号检查:

  ```python
  codes = [ws.cell(1, c).value for c in range(1, ws.max_column+1)]
  assert len(codes) == len(set(codes)), f"重号: {[c for c in codes if codes.count(c)>1]}"
  ```
