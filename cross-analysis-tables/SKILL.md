# 交叉分析表格生成

## 核心流程

```
0. 确认分析需求 → 1. 读取数据 → 2. 识别题目类型 → 3. 计算交叉统计 → 4. 输出单Sheet汇总表
```

## 零、确认分析需求（必须）

**在读取数据之前，必须先读取数据结构，然后向用户确认交叉分析的维度和题目。**

### 步骤1：先读数据识别题目

先读取 Excel，识别所有题目并分类：
- **基本情况题**（受访者属性题，通常在问卷开头）：如培训类别、选择因素、了解渠道、支出水平、学员规模等。**特征**：题号通常编号靠前，问的是"你是怎样的"而非"你怎么看"。
- **非基本情况题**（态度/行为/评价题）：如满意度、质量问题、续费意愿、监管评价等。

### 步骤2：询问分组变量（仅限基本情况题）

**问题1的选项只包含基本情况题，不提供其他题目。**

```
问题1: 你想用哪些基本情况题作为分组变量？
  - A1 培训类别
  - A2 选择因素
  - A3 了解渠道
  - A4 年度支出
  - ...（其他基本情况题）
  - 以上全部（默认推荐）
```

### 步骤3：询问分析题目

**问题2的第一个选项必须是"除分组变量外的所有题目"。**

```
问题2: 你想分析哪些题目的交叉分布？
  - 除分组变量外的所有题目（默认推荐）
  - 仅特定题目：B1/B2/B3/...（请指定）
```

默认行为：如果用户未指定，则分组变量选全部基本情况题，分析题目选除分组变量外的全部其他题目。

### 基本情况题的识别方法

读取数据后，按以下规则标注"基本情况题"：

1. **题目位置**：通常在问卷前部（前3-5道题），问受访者属性
2. **题目内容**：问"你是谁/你是什么样的"而非"你怎么看/你遇到什么"
3. **常见类型**：
   - 身份/分类题：培训类别、机构类型、学员规模等
   - 行为基础题：选择因素、了解渠道、消费水平等  
   - 人口统计题：所在区域、从业年限等
4. **判断题名**：如果题目标题含"类型"、"规模"、"渠道"、"因素"、"支出"等关键词，通常为基本情况题

**反例**（不是基本情况题）：满意度、续费意愿、推荐意愿、质量问题、监管评价、改进建议等。

## 一、数据读取

使用 Python + openpyxl 读取 Excel 数据：

```python
import openpyxl

wb = openpyxl.load_workbook("数据文件.xlsx", data_only=True)
ws = wb["整理"]  # 优先找"整理"sheet，否则用第一个数据sheet

# 典型宽表结构：第1行是短代码（Q, A1-1, A1-2...），第2行是完整题目文本
headers_code = [cell.value for cell in ws[1]]  # 短代码
headers_full = [cell.value for cell in ws[2]]  # 完整题目
data = []
for row in ws.iter_rows(min_row=3, values_only=True):
    data.append(dict(zip(headers_code, row)))
```

**数据清洗注意**：
- 跳过标识列为空或为表头标签行（如值为"整理"）的行
- 数值列可能有制表符前缀（`\t`），需 `str(val).strip()` 处理
- 多选题：有值=选中，空=未选
- 多选题的"其他"选项（如 A1-6 其他____）和对应的填空列（如 A1-7）中，填空列通常样本极小，分析时重点用选项列，填空列做备注参考即可

## 二、题目类型识别

| 类型 | 特征 | 处理方式 |
|------|------|----------|
| 单选 | 一列一个值，带选项前缀如"A.xxx" | 提取选项标签，统计频次分布 |
| 多选 | 多列编码如 Q-1, Q-2 | 统计各选项选中率 |
| 评分 | 数值列（如1-5分） | 计算均值 |
| NPS | 0-10分数值列 | 按规则映射为类别后计算分布 |
| 开放题 | 文本列 | 词频/主题归类 |

**选项标签提取**：单选题值通常为 `"A.选项文本"` 格式，作为原始标签保留。

## 三、统计计算规则

### 核心：占比存储为小数 + 单元格设为百分比格式

**占比类数据**在写入 Excel 时必须同时做两件事：
1. **值**：存储为小数（如 `0.2033` 表示 20.33%）
2. **格式**：设置 `cell.number_format = '0.0%'`，Excel 会自动显示为 `20.3%`

```python
cell.value = 0.2033            # 存储小数
cell.number_format = '0.0%'    # Excel 显示 20.3%
```

评分均值和 NPS 得分不属于百分比，不应用百分比格式，保留原始数值格式。

### 交叉分析占比计算（关键）

交叉分析表格中，每个数据单元格的含义是：**该选项的总体人群中，有多少比例同时属于该分组**。

```
行百分比 = 同时属于该分组且选中该选项的人数 / 该选项的总体选中人数
```

这是**行百分比**（选项总体的组内分布），而非列百分比（组内的选项分布）。
- 优点：由于多选题分组可能重叠（同一人同时属于多个组），行百分比避免了分母混淆
- 示例：8人选择了"按次/按周缴费"，其中2人同时属于"学科类"→ 行百分比 = 2/8 = 25.0%

```python
def row_percentage(group_mask, option_col, match_val, option_total):
    """计算：同时满足group和option的人数 / option总人数"""
    if option_total == 0:
        return 0.0
    hit = sum(1 for r in data_rows
              if group_mask(r) and matches(r, option_col, match_val))
    return round(hit / option_total, 8)  # 8位小数精度，配合number_format渲染
```

**注意区分**：
- **分组汇总行（Row 4-5）**使用列百分比：组人数 / 总样本数（传统占比）
- **分析题目行**使用行百分比：同时满足的人数 / 选项总人数（交叉分布）
- **评分均值行**使用组内均值：该组内所有有效评分的算术平均

### 评分题均值计算

```python
def calc_score_mean(rows, col):
    scores = [int(str(r.get(col)).strip()) for r in rows 
              if r.get(col) and str(r[col]).strip().isdigit()]
    return round(sum(scores)/len(scores), 2) if scores else 0
```

### NPS 计算

```python
def calc_nps(data, score_col):
    scores = []
    for r in data:
        v = str(r.get(score_col, '')).strip()
        if v.isdigit():
            scores.append(int(v))
    total = len(scores)
    promoters = sum(1 for s in scores if s >= 9)
    detractors = sum(1 for s in scores if s <= 6)
    nps = round((promoters - detractors) / total * 100) if total else 0
    return nps

# NPS 也可以按类别展示分布（推荐者/被动者/贬损者 → 占比，小数存储+百分比格式）
def calc_nps_distribution(data, score_col):
    # 映射规则：9-10→推荐者，7-8→被动者，0-6→贬损者
    # 返回各分类占比（小数，如 0.68 表示 68%）
    categories = {'推荐者(9-10分)': 0, '被动者(7-8分)': 0, '贬损者(0-6分)': 0}
    total = 0
    for r in data:
        v = str(r.get(score_col, '')).strip()
        if v.isdigit():
            s = int(v)
            total += 1
            if s >= 9: categories['推荐者(9-10分)'] += 1
            elif s >= 7: categories['被动者(7-8分)'] += 1
            else: categories['贬损者(0-6分)'] += 1
    return {k: round(v/total, 3) for k, v in categories.items()} if total else {}
```

## 四、交叉分析模式

### 模式A：单数据集内部交叉（全分组变量合一表）

**所有分组变量放在同一张表里作为列**，每个分组变量的每个选项各占一列。

**列结构完全由数据决定**：
- 列数 = 3（题号+题目/选项+总体） + 所有分组变量的选项总数
- 分组变量的每个选项（无论是单选的值还是多选的子列）各占一列
- 列的顺序按分组变量的出场顺序排列

```
列结构示意（数据驱动，不固定）:
题号 | 题目/选项 | 总体(N) | [分组1-选项A] | [分组1-选项B] | ... | [分组2-选项A] | [分组2-选项B] | ...
```

**行结构完全由数据决定**：
- 行数 = 表头区(5行) + 每题(题目标题行 + 选项行数) + 题间空行
- 单选题：N个选项 → N行（每个选项值一行）
- 多选题：M个子列 → M行（每个子列一行）
- 评分题：K个维度 → K行（每个维度一行）
- NPS：1行得分 + 3行分布 → 4行

**注意**：多选题受访者可能同时属于多个分组（如同一人既选了学科类又选了体育类），各组独立计算，共用一个总体的分母基数。

### 模式B：多数据集对比交叉

两份或多问卷中问同一件事的题目进行对比。需先建立题目映射（按"问的是同一件事"匹配，而非按编号）。

## 五、输出格式（单Sheet统一格式）

### 核心原则

1. **所有数据放在同一个 Sheet 中**，所有分组变量的选项作为列展开。
2. **结构和内容完全由问卷数据驱动**，不预设行数、列数、分组变量数量或选项数量。
3. **模版仅作格式参考**（样式、行/列语义、百分比的表示方式），具体内容必须根据实际问卷动态生成。

### 格式模板（示意，非固定）

以下展示的是**布局模式**，其中的题号、选项名称、数量都是示意性的。实际生成时每个字段都从问卷数据中动态提取。

```
Row 1: [空] | [空] | [空] | 【分组变量1】 |  (合并单元格跨所有子选项)  | 【分组变量2】 | ...
Row 2: [空] | [空] | [空] | 子选项1 | 子选项2 | 子选项3 | ...  | 子选项1 | 子选项2 | ...
Row 3: 题号 | 题目/选项 | 总体(N) | [分组1子选项A标签] | [分组1子选项B标签] | ... | [分组2子选项A标签] | ...
Row 4: Q   | 数量      | [总样本数] | [分组1子选项A人数] | [分组1子选项B人数] | ... | ...
Row 5: Q   | 占比      | 100.0% | [分组1子选项A占比] | [分组1子选项B占比] | ... | ...

[空行]

B1   | 【B1】题目标题  |       | 交叉值 | 交叉值 | ... | ...
B1   | A.选项1标签      |  [选项1总体占比] | 交叉值 | 交叉值 | ... | ...
B1   | B.选项2标签      |  [选项2总体占比] | 交叉值 | 交叉值 | ... | ...
...

[空行]

B2   | 【B2】题目标题  |       | 交叉值 | 交叉值 | ... | ...
...
```

> **关键**：分组变量有几个、每个分组变量有几个子选项、分析题有几道、每题有几个选项——全部由实际问卷数据决定。不要预设、不要硬编码、不要照抄模版的值。

> **格式说明**：上表展示的是 Excel 中的**显示效果**（`0.0%` 格式自动渲染）。实际存储值是小数（如 `0.203`），通过 `number_format='0.0%'` 实现显示。

### 各题型输出规则

| 题型 | 行标题(列B) | 列C(总体) | 数据列(占比) | 空值处理 |
|------|-----------|---------|------------|---------|
| 单选 | 原始选项标签("A.xxx")，按选项字母排序 | 总选中数 | 各组内占比(百分比数值) | 总体=0时填 "--" |
| 多选 | 原始选项标签("A.xxx")，按选项字母排序 | 总选中数 | 各组内选中率(百分比数值) | 总体=0时填 "--" |
| 评分(1-5) | 维度简称 | - | 各组均分(保留原始分数) | - |
| NPS | 分类标签(推荐者等) | 总人数 | 各组该分类占比(百分比数值) | 总体=0时填 "--" |
| NPS综合 | "NPS得分" | 总人数 | 各组NPS值(保留原始分数) | - |

### 格式规范

- **每个题目之间**：加一个空行分隔
- **题目段首行**：列A为题号(B1)，列B为题目标题(【B1】xxx)，其余列留空
- **占比数据**：存储为小数（如 `0.2033`），写入后设置 `cell.number_format = '0.0%'`，Excel 自动显示为 `20.3%`
- **"数量"行**：显示每个分组有多少样本，整数格式，不设百分比格式
- **"占比"行**：显示每个分组的样本占总体的比例，使用百分比格式。总体列存储 `1.0` → 显示为 `100.0%`
- **评分均值**：保留原始分数（1.0-5.0），不设百分比格式，保留1位小数
- **NPS得分**：保留原始整数分数（-100~100），不设百分比格式
- **NPS分布（推荐者/被动者/贬损者）**：各类占比使用百分比格式
- **选项排序**：单选/多选题的选项必须按选项字母前缀（A、B、C、D...）排序，不得按出现频次排序

## 六、Excel 输出规范

使用 openpyxl 生成格式化的 .xlsx 文件，**单个Sheet名称为"交叉分析"**：

**样式规范**：
- 分组表头行（Row 1-2）：合并同类分组的单元格，居中加粗
- 列标题行（Row 3）：蓝底白字，10号加粗，居中
- 数量和占比行（汇总区，紧接列标题之后）：浅灰底，居中
- 题目段标题行：浅蓝底，左对齐，加粗
- 数据行：浅蓝边框，居中
- 列宽：题号列10，题目/选项列40，数据列12-14（根据内容自适应）

**百分比单元格格式（关键）**：

所有占比类数据单元格必须设置 `number_format`：

```python
cell.value = 0.203       # 存储小数
cell.number_format = '0.0%'  # Excel 显示 20.3%
```

适用百分比格式的单元格：
- 汇总区"占比"行的所有数据列 + 总体列（总体列存储 `1.0`，显示为 `100.0%`）
- 单选题/多选题的每个选项行（所有数据列 + 总体列）
- NPS 分布的推荐者/被动者/贬损者行（所有数据列 + 总体列）

不适用百分比格式的单元格：
- 汇总区"数量"行：整数，不设格式
- 评分均值行：原始数值（如 `4.2`），不设格式
- NPS 得分行：原始数值（如 `62`），不设格式

**无需额外Sheet**：
- 不需要"索引"Sheet
- 不需要"关键发现"Sheet
- 所有内容在一张"交叉分析"Sheet中

## 七、常见陷阱

- **zip 截断 bug**：不要用 `zip(ALL_GROUPS, all_group_subcols)` 迭代分组列。`ALL_GROUPS`（如2-4个分组变量）和 `all_group_subcols`（如11-28个子列）长度不同，zip 以较短者为准，会导致大部分分组列无值。正确做法：预构建 `[(gdef, code), ...]` flat list，直接遍历该 list。
- **评分题误判为多选**：有些列的值为1-5数字（评分），而非选项标签文字。在分类题目类型前，务必抽样检查实际值。如果所有行都填了数字（非空），应归类为评分题（rating）展示均值。
- **照抄模版结构**：行数、列数、分组变量数量、选项数量必须由实际问卷数据推导，不得硬编码或照抄参考模版的值。
- **分母错误**：多选题用总选择次数做分母 → 应用分组内有效问卷总数
- **输出格式错误**：占比数据必须存储为小数（如 `0.203`），**并设置 `number_format='0.0%'`**。不能只设置值不设格式，也不能存储 `20.3` 后设 `0.0%`（会显示成 `2030.0%`）。NPS得分和评分均值不设百分比格式。
- **多Sheet错误**：不要每个分组变量建一个Sheet → 所有分组变量在同一Sheet中作为列
- **分组重叠**：多选题受访者可能属于多个分组 → 各组独立计算，不互斥
- **选项乱序**：单选/多选题选项必须按 A、B、C、D 字母顺序排列，不得按频次或出现顺序排序
- **忽略空值**：计算时未排除未作答 → 需明确分母是否包含未作答
- **数值前缀**：Excel导出数值可能带制表符 → 必须 strip 处理
- **基本情况题遗漏**：如果用户说"全部",默认分组变量=全部基本情况题
- **Row 1 分组标题合并错误**：当有多个分组变量时，ROW 1 必须为**每个分组变量独立合并其对应的列范围**，写入各自的标题（如 "G1 资质情况" 占 D1:H1、"G2 培训业务类型" 占 I1:N1）。**严禁**将所有分组列合并成一个单元格只写第一个变量名（如 D1:AE1 全合并写成 "A1 培训类别"），导致后续分组变量（A2/A3/A4 或 G2）在表头中完全不可见。
